#+TITLE: douo's Emacs Configuration
#+DESCRIPTION: 基于 Emacs 29+ 的个人配置，专为 macOS/Linux 设计，支持原生编译、Tree-sitter、Eglot 及丰富的 Org GTD 流程。

* 简介 (Intro)
基于 [[https://github.com/bbatsov/emacs.d/blob/965d39c245bdbe79e88dd228756a9cf621670ac0/init.el][bbatsov/emacs.d]] 裁剪定制的个人 Emacs 配置，采用 =straight.el= 管理包，支持原生的 Tree-sitter 和 Eglot LSP，并为知识管理（Org-roam）与远程开发（TRAMP）进行了深度优化。

* 核心依赖与环境要求 (Prerequisites)

为了保证所有配置项（如搜索、输入法、图标及 Markdown 预览等）正常工作，请在系统中安装以下依赖：

| 依赖名称 | 用途 (对应配置) | 安装/配置指引 |
|----------+----------------+---------------|
| [[https://github.com/BurntSushi/ripgrep][ripgrep]] | 模糊搜索 (=consult-ripgrep=) | =brew install rg= 或系统包管理器安装 |
| [[https://github.com/laishulu/macism][macism]] | 输入法中英文自动切换 (=sis=) | =brew install douo/macism/macism= |
| [[https://github.com/ryanoasis/nerd-fonts][Nerd Fonts]] | 状态栏及界面图标 (=nerd-icons=) | 安装 Nerd Font 字体，如 /SFMono Nerd Font/ |
| =locate= | 本地文件快速查找 (=consult-locate=) | Linux 下安装 =plocate= ；macOS 使用系统自带定位 |
| [[https://github.com/seagle0128/grip-mode][grip-mode]] | Markdown 实时预览 | 使用 [[https://github.com/immanelg/mdopen][mdopen]] 作为底层渲染引擎，已完全去除了 Python =grip= 依赖 |
| =mdopen= | Markdown 预览后台 | 通过 =cargo install mdopen= 安装，软链接至 =bin/mdopen= |
| *sshfs* | 远程图片挂载 (=image-dired=) | macOS： =brew install sshfs= (依赖 macFUSE fskit 后端)；Linux： =pacman -S sshfs= (依赖 FUSE，提供 =fusermount=) |
| *系统动态库* | 运行 Emacs 原生编译及图形支持 | 若遇到动态库报错，请安装： =brew install jpeg librsvg zlib libgccjit= |

* 本地安装与初始化 (Getting Started)

** 1. 基础克隆
#+begin_src shell
git clone https://github.com/douo/lims_dot_emacs.git ~/.emacs.d
#+end_src

** 2. 本地个性化配置 (=init-local.el=)
在 =~/.emacs.d/lisp/= 下创建 =init-local.el= 文件，用于存放特定机器的配置（此文件已被 =.gitignore= 忽略，不会提交到远程）：
#+begin_src elisp
;;; init-local.el -*- lexical-binding: t; -*-
;; 例如设置本地特定的网络 IP、跳板机、或者自定义字体
#+end_src

** 3. 初始化 Tree-sitter
在 Emacs 启动后，手动运行一次以下命令以自动下载并安装所有主流语言的 Tree-sitter 语法解析器：
#+begin_src elisp
M-x treesit-auto-install-all
#+end_src

** 4. 环境变量配置
本配置要求当前主机的 Shell 中暴露出以下环境变量，用来定制文件路径及 Conda 环境：
- =WRITING_HOME= : 个人写作/文档根目录
- =GTD_HOME= : GTD/日程事务管理根目录
- =CONDA_EXE= : Miniconda/Anaconda 可执行文件路径 (用于 =conda.el=，例如 =/opt/homebrew/bin/conda=)

在 GUI 下，配置会自动通过 [[https://github.com/purcell/exec-path-from-shell][exec-path-from-shell]] 将你终端 Shell 的环境变量无缝同步给 Emacs。

* 性能优化：启动预编译 (Precompilation)

虽然配置采用了懒加载（Lazy Loading），但在升级包或大量修改配置后，部分包在首次加载时可能仍会触发原生的 JIT 编译（Native Compilation），引起短时间卡顿。推荐通过配置脚本进行预热编译：

- *查看帮助*：
  #+begin_src shell
  emacs -Q --batch -l scripts/precompile.el -- --help
  #+end_src
- *演练预览*（不生成缓存，只查看待处理文件）：
  #+begin_src shell
  emacs -Q --batch -l scripts/precompile.el -- --dry-run
  #+end_src
- *执行一键编译*（产物生成于 =eln-cache/= 目录）：
  #+begin_src shell
  emacs -Q --batch -l scripts/precompile.el
  #+end_src
- *强行重新生成*：
  #+begin_src shell
  emacs -Q --batch -l scripts/precompile.el -- --force
  #+end_src

* 核心功能深度定制 (Key Features)

** 1. Markdown/xwidget 预览与键盘选择
Markdown 文件通过 =grip-mode= 调用 =mdopen= 渲染，并优先使用 Emacs 内置 xwidget WebKit 预览。预览 buffer 里的普通导航键默认映射到 xwidget 原生滚动，避免网页 Selection 和滚动位置出现两个不同步的隐藏光标。

*** 键盘光标与选择
在 xwidget 预览 buffer 中：
1. 按 =C-SPC= 进入 =Caret= 模式，只显示网页内光标，不立即选择文本。
2. 在 =Caret= 模式下，使用 =C-f= / =C-b= / =C-n= / =C-p= / =C-a= / =C-e= / =M-f= / =M-b= 等移动网页光标。
3. 再按一次 =C-SPC= 进入 =Select= 模式，从当前光标位置开始扩展选择。
4. 按 =M-w= 复制当前网页选择并退出；按 =C-g= 取消选择并退出。

*** TRAMP 远程 Markdown 预览
远程 Markdown 文件不会在远端启动 =mdopen= 。配置会在本地为每个预览创建独立临时目录，将远程文件内容写入其中，再从本地启动 =mdopen --reload= 。这样既能实时刷新当前预览，又避免 =mdopen= 监听整个系统临时目录，导致无关临时文件变化触发 xwidget 页面重载并清空网页选择。

按 =q= 或 =Q= 关闭 xwidget 预览时，会停止对应源 buffer 的 =grip-mode= 预览进程，并清理该预览的临时目录。

** 2. 远程图片流式浏览优化 (SSHFS 自动挂载)
*** 痛点与背景
默认情况下，内置的 =image-dired= 无法直接预览 TRAMP 远程路径下的图片（因为底层外部 =convert= 进程是本地运行的，无法识别远程协议路径）。如果通过 Advice 将其改为 TRAMP 远程调用或同步复制原图，又会因 =image-dired= 并发发起多个网络请求而导致 TRAMP SSH 隧道连接死锁（Emacs 100% CPU 卡死），或由于网络传输导致界面严重卡顿。

*** 解决方案与设计
相关实现已模块化至 =lisp/init-image-dired.el= ，并对 =image-dired-show-all-from-dir= 进行 Advice 包装拦截：
1. *检测路径*：若传入的是远程 TRAMP 路径，自动在后台静默通过 SSHFS 挂载远程目录至本地 =<挂载基目录>/image-dired-<host>-<hash>= 。
2. *本地渲染*：将挂载点作为本地目录传给 =image-dired= 运行，利用系统缓存技术流畅快速生成缩略图，且完全规避了 TRAMP 并发死锁。
3. *自动清理*：当你关闭（Kill）缩略图 buffer 或按 =q= 退出时，Emacs 会自动在后台异步卸载该挂载，并清理可能残留的孤儿 =sshfs= 进程；挂载点目录在 macOS 由系统随卸载移除、在 Linux 由模块删除（见下方“跨平台行为”），避免本地堆积垃圾。
4. *退出兜底*：由于 =kill-buffer-hook= 在退出 Emacs 时不会触发，模块额外在 =kill-emacs-hook= 上同步卸载所有活动挂载；并在每次启动时扫描挂载基目录，清理上次会话异常退出（或崩溃）遗留的挂载，确保不会跨会话累积。是否已挂载通过解析 =mount= 表判断（而非 =file-directory-p= ，后者在挂载僵死时会因 =EPERM= 误判为未挂载，从而在同一点叠加挂载、导致卸载后目录仍残留）。
5. *手动清理*：随时可执行 =M-x my-image-dired-sshfs-cleanup-stale= 一键卸载并删除挂载基目录下所有遗留（含僵死）的 image-dired 挂载。

*** 跨平台行为
挂载/卸载的差异全部收敛在 =lisp/init-image-dired.el= 的平台辅助函数中，按 =system-type= 自动分发：

| 维度 | macOS | Linux | 其它系统 |
|------+-------+-------+----------|
| 挂载基目录 | =/Volumes= | =$XDG_RUNTIME_DIR= （回退 =/tmp= ） | 不挂载 |
| 挂载后端 | macFUSE =fskit= ＋ =volname= | 标准 FUSE（最小参数，需先 =mkdir= 挂载点） | — |
| 卸载命令 | =diskutil unmount force= （可卸僵死挂载并自动删目录） | =fusermount3 -u= ／ =fusermount -u= ＋手动删目录 | — |

在不支持的系统上，Advice 会直接回退到 =image-dired= 的原始行为，不做任何挂载。

*注（macOS 权限）*：=/Volumes= 归 =root= 所有，普通用户无写权限，因此挂载点目录由系统（fskit）创建、并在 =diskutil unmount= 成功时由系统一并移除——模块不会（也无法）在 =/Volumes= 下手动 =mkdir= / =rmdir= 。判断是否已挂载一律通过 =mount= 表，而非 =file-directory-p= （后者在挂载尚未完成或僵死时会误判，把空目录当成挂载结果）。Linux 下挂载基目录归当前用户所有，则由模块自行创建与删除空目录。

** 3. 双系统 Org-roam 与浏览器捕获联动
通过 =org-protocol= 机制，可以实现一键从 Chrome 浏览器抓取网页标题、链接及选中文本并直接保存至 Emacs =org-roam= 库中。

*** 步骤 A：注册系统 Scheme-Handler
- *macOS*：将 =assets/roam/macOS/OrgProtocolClient.app= 复制到目录 =/Applications= 下。双击手动运行一次，以完成 =org-protocol://= 协议的系统注册。
- *Linux*：
  1. 复制 =assets/roam/linux/org-protocol.desktop= 到 =~/.local/share/applications/= 目录。
  2. 运行终端命令注册 Scheme-Handler：
     #+begin_src shell
     xdg-mime default org-protocol.desktop x-scheme-handler/org-protocol
     #+end_src

*** 步骤 B：配置 Chrome Shortkeys 扩展
在 Chrome 浏览器中安装 *Shortkeys* 插件，并按需添加以下快捷键规则：

- *网页捕获 (To Read Inbox)*:
  - *macOS* (快捷键 =ctrl+cmd+c=, 协议使用 =gtd-capture://=)：
    #+begin_src js
    location.href='org-protocol://gtd-capture://r/'+
      encodeURIComponent(location.href)+'/'+
      encodeURIComponent(document.title)+'/'+
      encodeURIComponent(window.getSelection())
    #+end_src
  - *Linux* (快捷键 =ctrl+alt+c=, 协议使用 =gtd-capture:/=)：
    #+begin_src js
    location.href='org-protocol://gtd-capture:/r/'+
      encodeURIComponent(location.href)+'/'+
      encodeURIComponent(document.title)+'/'+
      encodeURIComponent(window.getSelection())
    #+end_src

- *网页知识归档 (org-roam-protocol)*:
  - (快捷键 =ctrl+cmd+o= 或 =ctrl+alt+o=)：
    #+begin_src js
    location.href =
        'org-protocol://roam-ref?template=r&ref='
        + encodeURIComponent(location.href)
        + '&title='
        + encodeURIComponent(document.title)
        + '&body='
        + encodeURIComponent(window.getSelection())
    #+end_src

*** 步骤 C：禁用 Chrome 外部链接弹窗警示
在点击 Shortkeys 或书签栏捕获时，Chrome 默认会弹窗询问是否打开外部程序，可通过以下设置禁用该弹窗：
- *macOS*：终端执行
  #+begin_src shell
  defaults write com.google.Chrome URLAllowlist -array-add -string 'org-protocol://*'
  #+end_src
- *Linux*（写入相应浏览器的 Policy JSON 策略中）：
  - Arch Google-Chrome: =/etc/opt/chrome/policies/managed/org-protocol.json=
  - Arch Chromium: =/etc/chromium/policies/managed/org-protocol.json=
  - Ubuntu Chromium: =/etc/chromium-browser/policies/managed/org-protocol.json=
  - 策略内容定义如下：
    #+begin_src js
    {
      "AutoLaunchProtocolsFromOrigins": [
        {
          "allowed_origins": [ "*" ],
          "protocol": "org-protocol"
        }
      ]
    }
    #+end_src
  - (注：如果文件没有写入权限，可使用 =sudo chmod 444 <path>=)
  - 通过 Chrome 内部链接 =chrome://policy/= 可以检查策略是否正常生效。
