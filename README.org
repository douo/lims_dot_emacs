** 启动性能与预编译

懒加载可以减少 Emacs 启动时间，但某些包第一次打开时仍可能现场 native compile，表现为短时间卡顿。
需要预热时，可以手动运行预编译脚本，把已有包和本地 =lisp/*.el= 提前编译成 =.eln= 缓存。

查看帮助：
#+begin_src shell
emacs -Q --batch -l scripts/precompile.el -- --help
#+end_src

先查看本次会处理多少文件，不实际生成缓存：
#+begin_src shell
emacs -Q --batch -l scripts/precompile.el -- --dry-run
#+end_src

列出计划预编译的文件：
#+begin_src shell
emacs -Q --batch -l scripts/precompile.el -- --dry-run --list
#+end_src

执行预编译：
#+begin_src shell
emacs -Q --batch -l scripts/precompile.el
#+end_src

强制重新生成 =.eln=：
#+begin_src shell
emacs -Q --batch -l scripts/precompile.el -- --force
#+end_src

注意事项：
- 脚本只启动独立的批处理 Emacs 进程，不连接、不重载当前正在使用的 Emacs。
- 预编译产物写入 =eln-cache/=，不会生成本地配置的 =.elc=。
- 更新包、切换 Emacs 版本、或者大量修改配置后，建议手动运行一次。
- 这只能减少 Lisp/native-comp 首次加载成本；外部服务启动、网络请求、Tree-sitter grammar 下载、vterm 动态模块编译等仍需要单独处理。

* 远程图片预览优化 (SSHFS 自动挂载)

** 上下文与痛点
当通过 Emacs TRAMP 机制打开远程主机上的图片目录（例如使用 =image-dired= 浏览图片）时，默认情况下 Emacs 并不能直接生成缩略图（因为它会尝试在本地运行 =convert= 转换远程路径，导致路径无法被本地程序识别而报错失败）。在之前的第一版尝试中，我们通过 Advice 拦截了该过程，将其改为通过 TRAMP 进行远程转换（或将文件同步复制到本地）。但由于 =image-dired= 在渲染缩略图时默认会并发启动多个 =convert= 任务，这在 TRAMP 机制下会产生并发的 SSH 隧道请求，从而频繁触发 TRAMP 链接死锁（表现为 CPU 100% 占用，Emacs 界面完全卡死）；而采用同步复制又会因为网络传输阻塞 Emacs 主线程，导致界面严重卡顿。

** 解决方案与权衡

为了解决死锁并提升浏览性能，对比了以下方案：
1. *继续使用 TRAMP 异步处理*
   - *缺点*：无法根治死锁问题，Emacs 官方 TRAMP 机制对并发短连接子进程的稳定性存在隐患。
2. *远程执行 convert 并拉回缩略图*
   - *缺点*：远程主机可能未安装 ImageMagick 依赖；且多次单张拉取网络往返开销巨大，浏览几十张图会产生明显的等待感。
3. *SSHFS 自动挂载方案 (最终选择)*
   - *优点*：通过 macOS =macFUSE= 的 =fskit= 后端将远程目录挂载为本地目录。Emacs 在使用 =image-dired= 时，操作的是本地文件路径（=/Volumes/image-dired-...=），规避了 TRAMP 多线程死锁。同时利用 macOS 系统级缓存和 SSHFS 本身的长连接重连机制，极大地提升了加载速度与可靠性。
   - *自动化流程*：
     - 拦截 =image-dired-show-all-from-dir= 参数。
     - 若为远程 TRAMP 路径，自动在后台静默挂载 SSHFS 到 =/Volumes/image-dired-<host>-<hash>=。
     - 本地执行缩略图浏览。
     - 当杀死（Kill）缩略图 buffer 或按 =q= 退出时，自动在后台异步执行卸载（=umount=），释放挂载点。

** 系统依赖
实现此方案需要满足以下系统依赖：
1. 安装 *sshfs*（macOS 下包含 macFUSE 支持）：
   #+begin_src shell
   brew install sshfs
   #+end_src
2. 确保 Emacs 正常运行所需的相关 Homebrew 动态库（若在系统更新后遇到 launch 报错请安装）：
   #+begin_src shell
   brew install jpeg librsvg zlib libgccjit
   #+end_src
